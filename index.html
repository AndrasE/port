Crud

Crud, auth and fire

I wanted to create a simple CRUD (create, read, update, delete) React App with a backend. I thought try something new and ended up using Firebase for this. Once I archieved my goal, I wanted to experiment a little bit further and add authentication to the project so only registered users can interact with the database. I used firabse for authentication and also a context-provider & protected routes. If a visitor arrive on the landing-site an object gets created with guest user set to be true. Unless the authentication has taken place its remains true and don`t let user go any further gets redirected to login/register. On firebase end its also set that only registered user can manipulate the database. You can read about this project a lot more in details by going to crud and click on the logo and read more. I wanted to keep this project as simple as possible. If i would have more time to spend on it I would have add sessions(cookies) to it and also include that each user only able to interact with notes that is created by them and not a the whole database. For now, Im happy with the result, it required me to learn a lot.

Secrets

Our secrets!

It was a barebone project using MongoDB in the back with simple crud operations. Using EJS the repetations of code got reduced. Added authentication and session cookies to the project. Users only can interact with their own "secrets", but able to read all. After authentication sessions let users remain logged in unless the cookie expires or browiser get exited. You can aslo read more in details at the about/read more section of the page. Looking back now Im fairly happy with the results, altough I think this project would be more suitable to using React; too many redirects for my liking. Regarless Im proud of it.
